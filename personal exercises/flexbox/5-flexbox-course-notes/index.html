lesson 8-1 (wrapping up)


in this example we set the width of "home", "search" and "logout" to 1000 pixels,

However Flexbox does not allow the container's children to have bigger total width than the container itself.
This results in the children having lower width then we set in the css ruleset.
This is because of flex-wrap, which has a default value of nowrap.


.navbar {flex-wrap: nowrap;}
+
.navbar div {width: 1000px;}


Home
Search
Logout


lesson 8-2 (wrapping up)


(read 8-1 before continuing)


Here we have set flex-wrap to wrap, thus the children's width is being respected
and when the total width of the children gets bigger than that of their parent container
the children that can not fit within the container get pushed on a new row.


.navbar {flex-wrap: wrap;}
+
.navbar div {width: 1000px;}

lesson 9 (flex, grow, shrink, basis)



flex is shorthand for: flex-grow, flex-shrink, flex-basis



when you write (flex: 1) or (flex: 1 1 0), you essentially write this:

flex-grow: 1;
flex-shrink: 1;
flex-basis: 0;



flex-grow
allows the child to grow, when it's container increases in size

flex-shrink
allows the child to shrink, when it's container decreases in size

flex-basis
sets the width of the child

lesson 10-2 (order)


now we are going to make "search" appear at the very end, without modifying a single line of html code



.search {order: 1;}

1. Home
2. Search
3. Logout


Now you may be asking, why does "order: 1" make the ".search" div go last place?
because by default all elements have an order value of 0
and since 0 is before 1, home and logout go first and search ends up last
if you want ".search" to be first you can give it an order value of -1
